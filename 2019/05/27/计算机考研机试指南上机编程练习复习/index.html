2019年夏令营机试编程复习常识及入门

1. 常识

1.1 评判结构

  1. Accepted: 正确

  2. WA: wrong answer

  • 考虑代码健壮性比如边界条件，溢出等
  • 重新设计算法

  1. Presentation Error: 格式错误

  2. TLE: Time Limit Exceeded

  • 出现死循环，查看边界条件
  • 设计的算法的复杂度比较高，优化算法
  1. Runtime Error: 运行时错误
  • 访存是否错误，如数组下标越界
  • 除以0
  • 递归过深导致了栈溢出

  1. compile Error: 编译错误

  2. Memory Limit Exceeded: 内存超限，空间复杂度过高，或出现死循环

  3. Output Limit Exceeded: 输出超出限制

  • 可能是忘记了关闭调试信息

1.2 复杂度估计

1s的时间限制：复杂度不能超过百万级别：100,0000.

比如O(n2)O(n^2)O(n2), n<3000.

空间限制：32MB,不能超过使用32M的内存。
